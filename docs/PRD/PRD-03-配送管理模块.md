# PRD-03 配送管理模块

**页数**：约 7 页

---

## 1. 模块概述

配送管理模块覆盖配送任务分配、路线管理、状态流转、在途可视化、异常处理及配送绩效，实现配送调度智能化、在途可追踪。

---

## 2. 功能清单

| 功能编号 | 功能名称 | 优先级 | 说明 |
|----------|----------|--------|------|
| D-01 | 配送任务分配 | P0 | 智能/手动分配任务 |
| D-02 | 配送状态流转 | P0 | 待接单→配送中→已签收/异常 |
| D-03 | 路线管理 | P0 | 路线维护与绑定 |
| D-04 | 在途可视化 | P1 | 地图展示在途与 ETA |
| D-05 | 异常处理 | P1 | 异常登记与处理 |
| D-06 | 配送绩效 | P2 | 准时率、签收率统计 |

---

## 3. 功能详细说明

### 3.1 配送任务分配（D-01）

**角色**：配送调度员

**分配方式**：
- **手动分配**：选择订单、选择配送员、选择路线
- **智能分配**（P1）：按路线、运力、时效自动推荐

**操作流程**：
1. 进入「配送管理」→「待配送订单」
2. 筛选：区域、期望送达时间
3. 选择订单（可多选）
4. 点击「分配」
5. 选择配送员、路线
6. 确认 → 任务推送至配送员

**业务规则**：
- 配送员需在路线覆盖范围内
- 同一配送员同一时段任务数有上限（可配置）
- 分配后订单状态变为「待接单」

---

### 3.2 配送状态流转（D-02）

**状态定义**：

| 状态 | 说明 | 操作人 |
|------|------|--------|
| 待接单 | 已分配，待配送员接单 | - |
| 已接单 | 配送员确认接单 | 配送员 |
| 配送中 | 已取货，配送中 | 配送员 |
| 已签收 | 客户已签收 | 配送员 |
| 异常 | 无法送达等异常 | 配送员/调度员 |

**流转规则**：
- 待接单 → 已接单（配送员接单）或 待接单（配送员拒单，回池）
- 已接单 → 配送中（配送员开始配送）
- 配送中 → 已签收（正常）或 异常（异常登记）
- 异常 → 已签收（补送成功）或 取消（取消配送）

**拒单**：配送员可拒单，需填写原因，任务回待配送池

---

### 3.3 路线管理（D-03）

**字段**：路线编码、路线名称、覆盖区域、描述、关联配送员

**功能**：增删改查、路线与配送员绑定、路线与区域绑定

**分配逻辑**：分配任务时，按订单收货地址匹配路线，推荐该路线下的配送员

---

### 3.4 在途可视化（D-04）

**角色**：配送调度员

**功能**：
- 地图展示：配送员实时位置、在途订单、客户地址
- 预计送达时间（ETA）：基于距离、路况估算
- 配送员位置需配送员端 App 上报

**技术**：地图 API（高德/百度）、WebSocket 实时推送

---

### 3.5 异常处理（D-05）

**异常类型**：客户拒收、地址错误、联系不上、商品损坏、其他

**操作流程**：
1. 配送员/调度员登记异常，选择类型、填写备注
2. 异常订单进入「异常处理」列表
3. 调度员处理：安排重送、联系客户、取消订单等
4. 处理完成，更新订单状态

**客户投诉**：独立入口，可关联订单，分配处理人

---

### 3.6 配送绩效（D-06）

**指标**：
- 准时率：按时送达订单数 / 总订单数
- 签收率：成功签收数 / 总订单数
- 投诉率：被投诉次数 / 总订单数

**统计维度**：按配送员、按路线、按日期

---

## 4. 页面原型说明

### 4.1 待配送订单

- 列表：订单号、客户、地址、商品、期望送达、状态
- 操作：批量分配、单个分配

### 4.2 配送任务看板

- 按状态分栏：待接单、配送中、已签收、异常
- 卡片展示：订单摘要、配送员、路线、状态、操作

### 4.3 在途地图

- 地图中心：配送员位置
- 标记：客户地址、预计送达时间
- 侧边栏：在途订单列表

## 5. 接口设计（概要）

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/delivery/orders | GET | 配送订单列表 |
| /api/delivery/orders/{id}/assign | PUT | 分配任务 |
| /api/delivery/orders/{id}/status | PUT | 更新状态 |
| /api/delivery/routes | GET/POST | 路线管理 |
| /api/delivery/persons | GET/POST | 配送员管理 |
| /api/delivery/tracking | GET | 在途位置（WebSocket） |
