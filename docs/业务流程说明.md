# 生鲜超市物流配送系统 - 业务流程说明

## 一、用户下单流程

用户浏览商品 → 加入购物车 → 结算 → 创建订单 → 扣减库存

```mermaid
flowchart TD
    A[用户浏览商城] --> B[加入购物车]
    B --> C[进入购物车]
    C --> D[选择商品/修改数量]
    D --> E[去结算]
    E --> F{已登录?}
    F -->|否| G[提示先登录]
    F -->|是| H[提交订单]
    H --> I{库存充足?}
    I -->|否| J[返回库存不足]
    I -->|是| K[创建订单]
    K --> L[扣减商品库存]
    L --> M[清空购物车]
    M --> N[下单成功]
```

**关键步骤：**
- 购物车数据存于前端 localStorage
- 下单时校验：商品存在、库存充足、用户已登录
- 事务内完成：创建订单 + 写入订单明细 + 扣减库存
- 订单初始配送状态：`PENDING_ACCEPT`（待接单）

---

## 二、配送流程

管理员分配配送资源 → 更新配送状态

```mermaid
flowchart TD
    A[用户下单完成] --> B[订单状态: 待接单]
    B --> C[管理员进入配送管理]
    C --> D[点击分配]
    D --> E[选择配送员]
    E --> F[选择配送路线]
    F --> G[选择收货地址]
    G --> H[确认分配]
    H --> I[订单状态: 待接单]
    I --> J[管理员点击改状态]
    J --> K{选择新状态}
    K --> L[配送中]
    K --> M[已签收]
    K --> N[异常]
    L --> O[配送员配送]
    O --> M
    N --> P[填写异常备注]
```

**配送状态流转：**
```
PENDING_ACCEPT(待接单) → IN_DELIVERY(配送中) → DELIVERED(已签收)
                    ↘ EXCEPTION(异常)
```

## 三、采购流程

库存预警 / 手动申请 → 采购申请 → 审批 → 通过/驳回

```mermaid
flowchart TD
    subgraph 发起采购
        A1[仓储管理-库存预警] --> A2[点击发起采购]
        A3[采购管理-新建申请] --> A4[填写商品/数量/供应商]
        A2 --> A5[选择供应商+数量]
        A5 --> A6[提交采购申请]
        A4 --> A6
    end
    
    A6 --> B[采购申请状态: PENDING]
    B --> C[审批员进入待我审批]
    C --> D{审批操作}
    D --> E[通过]
    D --> F[驳回]
    E --> G[状态: APPROVED]
    F --> H[状态: REJECTED]
    G --> I[可进行入库等后续操作]
```

**采购申请来源：**
1. **仓储发起**：库存低于安全库存时，在仓储管理页点击「发起采购」
2. **采购页发起**：在采购管理「新建申请」中手动创建

**审批权限：** ADMIN、APPROVER 角色可审批

---

## 四、售后流程

用户申请售后 → 管理员处理

```mermaid
flowchart TD
    A[用户下单并完成] --> B[进入售后页面]
    B --> C[查看订单列表]
    C --> D[选择订单商品]
    D --> E{该商品已有售后?}
    E -->|是| F[显示: 售后申请中]
    E -->|否| G[点击申请售后]
    G --> H[填写售后原因]
    H --> I[提交申请]
    I --> J[售后状态: PENDING]
    
    J --> K[管理员进入售后管理]
    K --> L[查看售后列表]
    L --> M[点击改状态]
    M --> N{选择新状态}
    N --> O[PROCESSING 处理中]
    N --> P[CLOSED 已关闭]
    N --> Q[REJECTED 已驳回]
```

**约束：** 同一订单明细（orderItemId）在 PENDING 或 PROCESSING 状态下不可重复申请

---

## 五、商品评价流程

用户下单后对商品进行评价

```mermaid
flowchart TD
    A[用户完成下单] --> B[进入售后页面]
    B --> C[查看订单商品列表]
    C --> D{该商品已评价?}
    D -->|是| E[显示: 已评价]
    D -->|否| F[点击提交评价]
    F --> G[选择评分 1-5 星]
    G --> H[填写评价内容]
    H --> I[提交评价]
    I --> J[评价写入商品评论表]
```

**约束：** 仅下单用户可评价，且每个商品每个用户只能评价一次（前端根据 commented-by-user 判断）
