# 生鲜超市物流配送系统 - 用户调研报告

**版本**：v1.0  
**日期**：2025年3月  
**调研对象**：生鲜零售企业、配送站点、仓储管理人员、系统管理员

---

## 一、调研概述

### 1.1 调研目的

- 了解生鲜零售企业在采购、仓储、配送环节的痛点
- 明确各角色用户的核心诉求与使用场景
- 为系统功能设计提供依据

### 1.2 调研方法

- 深度访谈：5家生鲜零售企业、3个配送站点
- 问卷调查：回收有效问卷 120 份
- 现场观察：仓储作业、配送调度流程

### 1.3 调研对象画像

| 角色 | 人数占比 | 典型场景 |
|------|----------|----------|
| 采购员 | 15% | 发起采购申请、跟踪到货 |
| 仓储管理员 | 25% | 入库、出库、盘点、库存预警 |
| 配送调度员 | 20% | 分配配送任务、监控在途 |
| 配送员 | 25% | 接单、配送、签收 |
| 系统管理员 | 15% | 权限配置、基础数据维护 |

---

## 二、核心发现

### 2.1 采购环节痛点

- **审批流程不透明**：采购申请提交后无法实时查看审批进度
- **供应商信息分散**：供应商资质、报价、历史合作记录难以统一管理
- **采购计划与库存脱节**：缺乏基于库存预警的自动采购建议

### 2.2 仓储环节痛点

- **库存预警滞后**：依赖人工盘点，安全库存阈值难以动态调整
- **出入库效率低**：纸质单据流转，易出错、难追溯
- **库位管理粗放**：大品类分区，缺乏精细库位与批次管理

### 2.3 配送环节痛点

- **调度依赖经验**：路线、运力分配主要靠人工经验，缺乏系统辅助
- **在途不可视**：配送员位置、预计送达时间无法实时获取
- **异常处理被动**：客户投诉、配送异常事后才知晓

### 2.4 系统管理痛点

- **权限粒度粗**：按角色一刀切，无法细粒度控制功能与数据
- **操作无审计**：关键操作无日志，责任难以追溯
- **基础数据维护繁琐**：商品、供应商、路线等需多系统维护

---

## 三、用户诉求汇总

| 诉求类型 | 具体描述 | 优先级 |
|----------|----------|--------|
| 采购审批可视化 | 申请、审批、驳回全流程可追踪 | P0 |
| 库存自动预警 | 低于安全库存自动提醒并建议采购量 | P0 |
| 配送智能调度 | 按路线、运力、时效自动分配任务 | P0 |
| 角色权限细粒度 | 支持功能权限、数据权限、审批权限 | P0 |
| 出入库电子化 | 扫码/录入替代纸质单据 | P1 |
| 在途可视化 | 配送员位置、ETA 实时展示 | P1 |
| 供应商档案管理 | 资质、报价、评级统一管理 | P1 |
| 操作审计日志 | 关键操作可追溯 | P2 |

---

## 四、结论与建议

1. **优先建设**：采购审批、库存预警、配送调度、角色权限四大核心能力
2. **分阶段落地**：一期完成审批流与预警，二期完善调度与权限
3. **移动端补充**：配送员端需独立 App 或小程序支持接单、签收
